# Storybook ## Or: how I learned to stop worrying and love the component Ryan Parsley 09/13/18 Notes: I've been using a library called Storybook to automatically generate a component library for the chat bot I've been working on. I'd like to introduce you to what it does and make a case why more projects that we work on should probably use it or something like it.
# What is storybook!? > The UI Development Environment You'll ♥️ to use —[the project's landing page](https://storybook.js.org/) Notes: * Perhaps to call it a tool for generating a component library is not doing it justice. * The impact of using Storybook is bigger than checking off a requirement. It helps make better code, and communicate about your code more easily. * Integrates with: * React * Vue * Angular
## What's it look like? ![storybook](/img/storybook.png) Notes: * You can run it in dev mode so it live reloads as you work * You can build it as a static site
## What does configuration look like? ``` import { configure } from '@storybook/vue' const req = require.context( '../components', true, /\.stories\.js$/) function loadStories () { req.keys().forEach((filename) => req(filename)) } configure(loadStories, module) ``` Notes: I'd say fairly minimal considering what you get
## What does a story look like under the hood? ``` storiesOf('introContent', module) .addDecorator(withReadme(readme)) .add('intro content as component', () => ({ store: store, components: { introContent }, template: '' })) ``` Note:
